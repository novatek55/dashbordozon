# Инструкция для Claude: анализ временного ряда

## Роль

Ты — аналитик данных. Твоя задача — получить массив числовых значений (временной ряд) и оценить **последнее значение**: оно в норме, требует внимания или критично.

## Входные данные

Пользователь передаёт:

- `data` — полный массив значений ряда (хронологический порядок, слева направо)
- `window` — размер окна наблюдения (по умолчанию 7, это количество последних значений для фокусного анализа)

Пример:

```
data = [44, 45, 45, 42, 41, 35, 36, 42, 41, 36, 34, 34, 37, 40, 33, 36, 35, 36, 28, 28, 27, 31, 32, 39, 40, 34, 33, 36, 37, 38]
window = 7
```

## Метод анализа

Применяй **все 5 методов** последовательно к последнему значению ряда (`val = data[последний]`). Каждый метод выносит вердикт: `НОРМА`, `ВНИМАНИЕ` или `ТРЕВОГА`.

---

### 1. Z-score (стандартное отклонение от среднего)

```
mean = среднее всех значений data
std  = стандартное отклонение всех значений data
Z    = (val - mean) / std
```

| |Z| | Вердикт |
|------|---------|
| < 1 | НОРМА |
| 1–2 | ВНИМАНИЕ |
| > 2 | ТРЕВОГА |

**Зачем:** показывает, насколько значение отклоняется от общего среднего в единицах стандартного отклонения. Простой и универсальный метод.

**Ограничение:** чувствителен к выбросам в самом ряде — один экстремальный выброс раздувает std и маскирует остальные аномалии.

---

### 2. Модифицированный Z-score (на медиане и MAD)

```
median = медиана всех значений data
MAD    = медиана абсолютных отклонений от медианы
Z_mod  = 0.6745 × (val - median) / MAD
```

| |Z_mod| | Вердикт |
|----------|---------|
| < 2.5 | НОРМА |
| 2.5–3.5 | ВНИМАНИЕ |
| > 3.5 | ТРЕВОГА |

**Зачем:** робастная версия Z-score. Медиана и MAD устойчивы к выбросам, поэтому метод лучше работает на «грязных» рядах с редкими экстремумами.

**Коэффициент 0.6745** — это обратное значение 75-го перцентиля нормального распределения, нужен для совместимости шкалы с классическим Z-score.

---

### 3. IQR-метод (межквартильный размах)

```
Q1  = 25-й перцентиль data
Q3  = 75-й перцентиль data
IQR = Q3 - Q1
нижняя_граница = Q1 - 1.5 × IQR
верхняя_граница = Q3 + 1.5 × IQR
```

| Условие | Вердикт |
|---------|---------|
| val в [нижняя, верхняя] | НОРМА |
| val за пределами | ТРЕВОГА |

**Зачем:** классический boxplot-метод. Не зависит от формы распределения. Хорошо ловит явные выбросы.

**Ограничение:** бинарный (норма/выброс), не имеет промежуточной зоны «внимание».

---

### 4. Bollinger Bands (полосы Боллинджера)

```
MA  = скользящее среднее за последние window значений
σ   = стандартное отклонение за тот же window
верхняя_полоса = MA + 2σ
нижняя_полоса  = MA - 2σ
```

| Условие | Вердикт |
|---------|---------|
| val внутри полосы | НОРМА |
| val за полосой | ВНИМАНИЕ |

**Зачем:** оценивает значение в контексте **локальной** (а не глобальной) волатильности. Если ряд недавно был спокойным, даже небольшое отклонение будет замечено. Пришёл из финансового анализа.

**Параметры:** ширина окна = `window`, множитель = 2 (стандартное значение).

---

### 5. Линейный тренд (последние N значений)

```
x = [0, 1, 2, ..., window-1]
y = последние window значений
slope = коэффициент наклона линейной регрессии (метод наименьших квадратов)
```

| slope | Вердикт |
|-------|---------|
| > 0.3 | НОРМА (рост) |
| от -0.5 до 0.3 | НОРМА (стабильно) |
| < -0.5 | ВНИМАНИЕ (снижение) |

**Зачем:** показывает **направление движения**, а не абсолютное положение. Значение может быть ниже среднего, но если тренд восходящий — ситуация улучшается.

**Пороги** (0.3 и -0.5) — настраиваемые. Адаптируй под свой контекст: для метрики, где нормальный разброс ±1 в день, пороги должны быть мягче.

---

## Формат ответа

Выведи результат в следующей структуре:

```
## Исходные параметры
- Последнее значение: {val}
- Среднее ряда: {mean}
- Медиана: {median}
- Стд. отклонение: {std}
- MAD: {mad}
- Перцентильный ранг последнего значения: {percentile}%

## Результаты по методам

| Метод | Значение | Порог | Вердикт |
|-------|----------|-------|---------|
| Z-score | ... | |Z| < 1 / < 2 / > 2 | НОРМА / ВНИМАНИЕ / ТРЕВОГА |
| Мод. Z-score | ... | |Z| < 2.5 / < 3.5 / > 3.5 | ... |
| IQR | ... | [нижняя, верхняя] | ... |
| Bollinger | ... | [нижняя_полоса, верхняя_полоса] | ... |
| Тренд | ... в день | > 0.3 / > -0.5 / < -0.5 | ... |

## Итоговый вердикт
{N} из 5 методов показывают НОРМА.

{Краткий вывод: нужно ли бить тревогу, и почему.
Обязательно укажи контекст — например, было ли недавнее падение и идёт ли восстановление.}
```

## Дополнительные указания

1. Если данных меньше, чем `window` — используй все данные и укажи это.
2. Перцентильный ранг = доля значений ряда ≤ val (в %).
3. Все числа округляй до 1–2 знаков после запятой.
4. Если пользователь передаёт свои пороги — используй их вместо стандартных.
5. При расчёте стандартного отклонения используй **популяционную** формулу (делить на N, не на N-1), если не указано иное.

Окно: 7
Оцени последнее значение.

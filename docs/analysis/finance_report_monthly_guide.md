# Инструкция: анализ Finance Report за месяц

## Как сгенерировать отчёт

```bash
# Текущий месяц vs предыдущий
python -m src.export_finance_report --month 2026-05 --compare 2026-04

# Только текущий месяц (без сравнения)
python -m src.export_finance_report --month 2026-05
```

Файл сохраняется в `exports/finance_report_YYYY-MM.md`.  
Шаблон итоговых выводов (Раздел 7) заполняется вручную после анализа.

---

## Порядок анализа

Открыть файл и пройти разделы по порядку. На каждом шаге — один вопрос: **«это нормально?»**

---

### Раздел 1: P&L — общий рост

Смотреть на три строки:

| Строка | Норма | Тревога |
|---|---|---|
| Чистая выручка Δ | рост | падение два месяца подряд |
| ИТОГО расходы MP Δ | растут медленнее выручки | растут быстрее выручки |
| ВАЛОВАЯ ПРИБЫЛЬ Δ | растёт | падает при росте выручки |

**Главный сигнал:** если выручка растёт, а валовая прибыль падает — расходы MP съедают рост.

---

### Раздел 2: Структура % — эффективность условий

Анализировать **изменение** долей, а не абсолютные значения.

| Метрика | Хорошо | Плохо |
|---|---|---|
| Комиссия % | стабильна или снижается | рост > 1 п.п. за месяц |
| Логистика % | снижается при росте выручки | растёт при стабильном миксе |
| Реклама % | снижается (органика растёт) | растёт при падении выручки |
| Деньги на счёт % | растёт | снижается при росте выручки |
| Валовая прибыль % | растёт | снижается 2+ месяца подряд |

**Ориентир валовой прибыли:** держать выше 25%. Ниже — нужен пересмотр цен или себестоимости.

**Если комиссия % выросла** — переходить к отчёту по артикулам: смотреть по каким схемам FBO/FBS больше продаж и какие ставки по категориям.

---

### Раздел 3: Юнит-экономика — что происходит с одним заказом

Убирает эффект объёма и показывает реальное изменение затрат.

| Метрика | Смотреть на |
|---|---|
| Средний чек | падает = продаём дешевле или изменился микс товаров |
| Логистика / заказ | растёт = больше дорогих регионов или переход на FBS |
| Реклама PPC / заказ | растёт = реклама теряет эффективность, поднять ставки или оптимизировать |
| Валовая прибыль / заказ | падает при стабильном чеке = растут затраты |

**Правило:** если средний чек падает, а валовая/заказ держится — бизнес масштабируется нормально (продаём больше дешёвых, но эффективных товаров). Если и чек и валовая падают одновременно — проблема.

---

### Раздел 4: Детализация расходов — найти аномалии

Смотреть только на строки с Δ > 30% или > 10 000 ₽ абс.

**Логистика:**
- `Обратная логистика` растёт — возвраты с доставкой, проверить возвраты по товарам
- `Обработка дропофф` / `Дропофф партнёр` резкий рост — изменилась схема отгрузки

**Реклама:**
- `Закрепление отзывов` или `Ускоренные отзывы` неожиданно выросли — проверить не запустили ли автоматически
- `Premium+ подписка` — фиксированная 24 990 ₽/мес, должна быть стабильной

**Прочие сервисы:**
- `Временное хранение партнёр` — если растёт, товары задерживаются на промежуточных складах; ускорить поставки
- `Доставка до ПВЗ` — если растёт, покупатели всё чаще выбирают ПВЗ вместо курьера (обычно норма)

> **📦 Особенность крупногабарита (SKU 210, 214):**  
> Для этих артикулов курьер Ozon не приезжает на забор — отгрузка только через дропофф-партнёра.  
> Следствия в отчёте:
> - `Обработка дропофф` и `Дропофф обработка партнёр` растут при активных продажах этих SKU — **это норма**, не аномалия.
> - `Штраф нерекомендованный слот` возникает именно по ним: не успели в рекомендованный слот дропофф-партнёра → Ozon штрафует за несвоевременную отгрузку.
> - **Действие:** при планировании поставок SKU 210/214 всегда заранее бронировать рекомендованный слот у дропофф-партнёра.

---

### Раздел 5: Возвраты — скрытые потери

**Ключевое:** `Комиссия с возвратов` — это деньги, которые Ozon **не возвращает** при возврате товара. Это прямые потери.

Формула реальных потерь от одного возврата:
```
Потери = цена_товара × ставка_комиссии
```

Пример: товар 10 000 ₽, ставка 48% → потери с одного возврата = 4 800 ₽.

| Сигнал | Действие |
|---|---|
| Возврат % > 5% | Смотреть отчёт по возвратам: какой товар, какая причина |
| Комиссия с возвратов растёт быстрее выручки | Аналогично |

---

### Раздел 6: Штрафы и компенсации

**Штрафы — полностью управляемая статья, норма = 0.**

| Тип штрафа | Причина | Как устранить |
|---|---|---|
| `нерекомендованный слот` | Поставка в FBO в нерекомендованное время | Выбирать рекомендованные слоты при планировании отгрузки |

**Компенсации от Ozon** — Ozon платит за потерянные/повреждённые товары на складе.
- Упали до нуля → проверить: нет ли потерь на складе, которые не зафиксированы (подать претензию через ЛК)
- Резко выросли → хорошо, Ozon признал ущерб

# Шаблон итогового отчёта (Раздел 7)

```markdown
## 7. Выводы — [YYYY-MM]

**✅ Хорошо:**
- [что выросло / улучшилось — с цифрами]
- [пример: Валовая прибыль +44% до 3.1M ₽, маржа 31.2% — лучший результат за квартал]

**⚠️ Внимание (мониторить):**
- [что ухудшается, но ещё не критично — с трендом]
- [пример: Ставка комиссии 45.1% → 48.9% (+3.8 п.п.) — нужен анализ по схемам FBO/FBS]

**🔴 Действие (сделать до следующего отчёта):**
- [конкретное действие + ответственный + дедлайн]
- [пример: Штраф нерекомендованный слот 51 038 ₽ → при следующей поставке выбрать рекомендованный слот]

**📌 Следующие отчёты:**
- [какой отчёт открыть чтобы разобрать сигнал из "Внимание"]
- [пример: Отчёт по артикулам — разбить комиссию по FBO/FBS и категориям]
```

---

## Правила оформления

1. **Только цифры с контекстом** — не «выручка выросла», а «выручка +37% до 9.9M ₽»
2. **Δ п.п. для процентов** — не «комиссия выросла с 45% до 49%», а «+3.8 п.п.»
3. **Не больше 3 пунктов** в каждом блоке — приоритизировать главное
4. **Действие = глагол + объект + срок** — «отключить рекламу 405-50 до 05.06», не «рассмотреть вопрос рекламы»
5. **Следующие отчёты** — всегда указывать куда идти дальше, не оставлять сигнал без адреса

# Дизайн: Monthly AI Report — Ежемесячный MD-отчёт для ИИ-анализа

**Дата:** 2026-06-02  
**Статус:** Approved

---

## Цель

Генерировать MD-файл с данными за календарный месяц по магазину Ozon для подачи в ИИ (Claude/ChatGPT) с целью получения конкретных рекомендаций по ценообразованию, акциям, рекламе и управлению остатками.

---

## Архитектура

Новый модуль `src/services/monthly_report.py` — единственная точка сборки отчёта. Он переиспользует уже существующие сервисные функции, не дублируя SQL.

Два способа использования:
1. **Endpoint** `GET /api/monthly-report?month=YYYY-MM` — отдаёт MD как `text/markdown`, браузер предлагает скачать файл
2. **CLI** `python -m src.export_monthly_report --month 2026-05` — сохраняет в `exports/monthly_report_YYYY-MM.md`

---

## Источники данных (переиспользование существующих функций)

| Секция отчёта | Источник |
|---|---|
| Магазин-итого | `get_finance_report_data()` → `kpi_summary` + `rows_map[*].total` |
| Товары: экономика | `get_accruals_comp_by_article()` → начисления/комиссии по offer_id |
| Товары: продажи | SQL из `load_stock_forecast_inputs()` — fact_order_items за месяц |
| Остатки и оборачиваемость | `load_stock_forecast_inputs()` → FBO/FBS + days_of_stock |
| Реклама | `get_advertising_report()` → расход/ДРР/воронка по кампаниям |
| Акции | Прямые запросы к `promo_actions` / `promo_products` |
| Рейтинг и отзывы | `seller_rating_history`, `reviews` за период |

## Структура MD-файла

### Шапка

```markdown
# Ozon Monthly Report — 2026-05
Generated: 2026-06-01  
Period: 2026-05-01 — 2026-05-31
```

### Секция 1: Магазин — итоги месяца

Агрегаты за месяц из finance report:

| Метрика | Значение | % от выручки |
|---|---|---|
| Выручка (gross) | ₽ | — |
| Возвраты | ₽ | % |
| Чистая выручка | ₽ | — |
| Комиссия Ozon | ₽ | % |
| Логистика | ₽ | % |
| Реклама | ₽ | % |
| Прочие расходы MP | ₽ | % |
| Итого расходы MP | ₽ | % |
| Валовая прибыль | ₽ | % |
| Заказов (шт.) | | |
| Возвратов (шт./%) | | |
| Рейтинг продавца | | |
| Новых отзывов | | |
| Средняя оценка | | |

### Секция 2: Товары

Все товары с ненулевой активностью за месяц (хотя бы одно из: заказы > 0, реклама > 0, остаток > 0, в акции).

Каждый товар — отдельный блок `### Товар: {name} [{offer_id}]`:

```markdown
### Товар: Название [OFFER-123]

#### Продажи
| Заказов шт. | Выручка ₽ | Ср. цена ₽ | Возвратов шт. | Возвратов % |

#### Экономика (начисления)
| Начислено ₽ | Комиссия ₽ | Логистика ₽ | Чистое ₽ | Маржа % |

#### Остатки
| FBO шт. | FBS шт. | Продаж/мес | Дней запаса | Статус |

Статусы: ⚠ заканчивается (<30д) | ❄ залёживается (>120д) | ✓ норма

#### Реклама
| Расход ₽ | Показы | Клики | CTR % | Заказы | ДРР % |

#### Акции
| Акция | Цена акц. ₽ | Макс. цена ₽ | В акции | Кандидат |
```

### Секция 3: Реклама — сводка

Итоговые таблицы по кампаниям:

```markdown
## Реклама — сводка

### По кампаниям
| Кампания | Тип | Расход ₽ | Показы | Клики | Заказы | ДРР % |

### Топ-5 по расходу
| Товар | Расход ₽ | Заказы | ДРР % |

### Топ-5 по ДРР (худшие — высокий ДРР)
| Товар | ДРР % | Расход ₽ | Заказы |
```

### Секция 4: Остатки и оборачиваемость

Сводная таблица только по товарам с отклонениями:

```markdown
## Остатки и оборачиваемость

### ⚠ Заканчиваются (< 30 дней)
| Товар | FBO | FBS | Дней запаса | Продаж/мес |

### ❄ Залёживаются (> 120 дней)
| Товар | FBO | FBS | Дней запаса | Продаж/мес |
```

### Секция 5: Акции

```markdown
## Акции

### Активные акции периода
| Акция | Дата начала | Дата конца | Товаров в акции |

### Товары-кандидаты (не вошли в акцию)
| Товар | Текущая цена ₽ | Макс. цена акц. ₽ | Разница % |
```

---

## Параметры генерации

- `month` — обязательный, формат `YYYY-MM`
- Товары без активности полностью пропускаются
- Числа: округление до 2 знаков для ₽, до 1 знака для %
- Кодировка файла: UTF-8

---

## Файловый вывод

- Путь: `exports/monthly_report_{YYYY-MM}.md`
- Папка `exports/` создаётся автоматически если отсутствует
